Problem Statement

Find Peak Element

Given an integer array nums, find a peak element and return its index. A peak element is one that is strictly greater than its neighbors. Assume nums[-1] = nums[n] = -∞. If multiple peaks exist, return the index to any peak. You must write an algorithm that runs in O(log n) time.

🏔️ Imagine walking a mountain range in fog. At any point you can only see the next step forward. If the ground rises ahead, keep going — a peak is further. If the ground drops ahead, you might be at or past a peak — turn back. You'll always find high ground.
Input
nums = [1, 2, 3, 1]
→ index 2 (value 3)
Input
nums = [1,2,1,3,5,6,4]
→ index 5 (value 6)
Edge: single element
nums = [1]
→ index 0 (trivially a peak)
Intuition + Hints
  • 1 Brute force O(n): Scan every element and check if it's greater than both neighbors. Works, but wastes the log-n opportunity.
  • 2 The slope insight: Pick mid. Compare array[mid] with array[mid+1]. If array[mid] > array[mid+1], the slope is falling — a peak exists at or before mid. If rising, a peak is strictly ahead.
  • 3 Why does it work? Because the problem guarantees nums[-1] = nums[n] = -∞. The array must eventually come back down, so any upslope necessarily leads to a peak.
  • 4 Convergence: Each iteration halves the search space. When left == right, that index is surrounded by lower values (or -∞ boundaries) — it is the peak.
Key Trick: You never need to compare array[mid-1]. One comparison (array[mid] vs array[mid+1]) determines the slope direction — and slope direction tells you which half to search.
Your Solution
find-peak-element.js
var findPeakElement = function(array) {
 let left = 0, right = array.length-1

  while(left < right){
     let mid = Math.floor((left + right)/2)
    if(array[mid] > array[mid+1]){
      right = mid
    }else{
      left = mid +1
    }
  }
  return left
};

⚠️ Gotchas & Common Pitfalls
  • Mid+1 Out of Bounds: while(left < right) ensures that mid will strictly be less than right, so array[mid+1] is mathematically guaranteed to be safe!
  • Returning left vs right: The loop breaks exactly when left == right, so they point to the same peak. Returning either one is completely fine.
⚖️ Trade-offs & Alternatives
  • O(log N) Binary Search (Current): O(1) Space. Leverages the mathematical guarantee that following an upward slope bounding a -∞ edge will always yield a peak.
  • O(N) Linear Scan: Trivial to write for(let i=0; i and check neighbors, but fails the strict interview requirement for logarithmic time.
