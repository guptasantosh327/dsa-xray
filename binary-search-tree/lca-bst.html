LCA of a Binary Search Tree

Step-by-step trace of lowestCommonAncestor — watch how the BST ordering lets you eliminate entire subtrees instantly. No counting, no post-order — just compare and go.

Pattern
🎯 Pre-order DFS + BST Split Detection
At every node: if both p and q are smaller → go left. If both larger → go right. Otherwise → you're at the split point = LCA. Return immediately.
Same pattern unlocks: #700 Search BST · #98 Validate BST · #236 LCA (General Tree)
📋 Problem Statement
BST Superpower: For every node, all values to the left are smaller and all values to the right are larger. This means one comparison tells you exactly where p and q live — without searching.

Given a BST and two nodes p and q that are guaranteed to exist in the tree, find their Lowest Common Ancestor — the deepest node that is an ancestor of both.

⚡ Because it's a BST (not just any tree), you don't need post-order count propagation. The ordering directly tells you when p and q split across a node — and that split point IS the LCA.

p=2, q=8 → split at root LCA = 6
p=2, q=4 → both left, split deeper LCA = 2
💡 Intuition + Hints — Three Possible Cases
1
Both p and q are smaller than root? Then both must be in the left subtree (BST guarantee). The LCA can't be here — go left. lowestCommonAncestor(root.left, p, q)
2
Both p and q are larger than root? Then both must be in the right subtree. Go right. lowestCommonAncestor(root.right, p, q)
3
Neither of the above? That means they're on opposite sides of root — OR one of them IS root. This is the split point. Root is the LCA! Return root immediately.
4
Why does case 3 handle "p === root"? If p.val === root.val, then p is NOT smaller than root (case 1 fails) and NOT larger (case 2 fails) → we hit case 3 → return root. Correct! p is an ancestor of q in this case.

Core insight: In a BST, the LCA is the first node where p and q "split" — one goes left, the other goes right (or one IS the current node). Before the split they travel together in the same direction. That traveling-together property is what the BST ordering gives you for free. No counting, no marking, no post-order needed.

⚡ BST vs General Tree LCA: The general tree version (#236) needs post-order + count propagation to find where p and q meet. The BST version is simpler: the ordering directly tells you the meeting point in O(log n). Same goal, completely different approach. The BST superpower at work again.
lcaBST.js
var lowestCommonAncestor = function(root, p, q) {
  // Case 1: both smaller → LCA must be left subtree
  if (p.val < root.val && q.val < root.val) {
    return lowestCommonAncestor(root.left, p, q)
  }
  // Case 2: both larger → LCA must be right subtree
  else if (p.val > root.val && q.val > root.val) {
    return lowestCommonAncestor(root.right, p, q)
  }
  // Case 3: split here (or one IS root) → root is the LCA
  else {
    return root    // ← this node is the LOWEST COMMON ANCESTOR
  }
};

🧠Mental Model — The Fork in the Road
▸
Think of descending a mountain path. You're walking down. At every fork, you check: are both destinations to the left? Go left. Both to the right? Go right. If they split here — THIS fork is the last common point. That's the LCA.
▸
The three conditions are exhaustive. "Both left" + "Both right" + "else" covers every possible case. The "else" also catches when root.val === p.val or root.val === q.val — node is an ancestor of itself.
▸
Why pre-order? You check root BEFORE recursing. The moment you hit the split case, you return immediately — no need to go deeper.
▸
O(log n) not O(n). Unlike the general tree LCA which must potentially visit all nodes, this BST version eliminates one half of the tree at every step — just like BST search.
⚖️ BST LCA vs General Tree LCA — Know the Difference
BST LCA (#235) — This problem
  • Uses BST ordering property
  • Pre-order traversal (top-down)
  • 3 cases: left / right / split-here
  • O(log n) time in balanced BST
  • No counting, no post-order needed
  • Returns immediately at split point
General Tree LCA (#236)
  • No ordering to exploit
  • Post-order traversal (bottom-up)
  • Count propagation + !lca guard
  • O(n) time — must visit all nodes
  • Uses closure variable to capture answer
  • Returns count up the tree
⬅️

Case 1: Both Left

p < root && q < root → BST guarantees both nodes are in left subtree. Skip right entirely. Recurse left.

➡️

Case 2: Both Right

p > root && q > root → BST guarantees both nodes are in right subtree. Skip left entirely. Recurse right.

🎯

Case 3: Split / Hit

Everything else: they split across root, OR one of them equals root. Either way, root is the deepest ancestor of both. Return root.

📐

No base case needed

The problem guarantees p and q exist. You'll always reach the split case before hitting null. The three cases are exhaustive.

⚠️Edge Cases
p IS ancestorOne node is an ancestor of the other: e.g. p=2, q=4 in tree rooted at 6. When you reach node 2 (p), its value is not smaller than itself, not larger, so case 3 fires — returns node 2. Correct! p is ancestor of q.
adjacentParent-child pair: Same logic as above. When you reach the parent, the split condition fires immediately since the child must be on one specific side (but parent doesn't satisfy "both smaller" or "both larger").
guaranteedBoth nodes guaranteed to exist: The problem guarantees p and q are in the tree. So you'll always hit the split case. No null checks needed inside the recursion.
iterativeCan be written iteratively: Replace recursion with a while(root) loop using the same three-case logic. Same O(log n) time, O(1) space instead of O(h) call stack.
✦ Clean solution — recursive + iterative

// Recursive — clean and readable
var lowestCommonAncestor = function(root, p, q) {
  if (p.val < root.val && q.val < root.val)
    return lowestCommonAncestor(root.left, p, q);
  if (p.val > root.val && q.val > root.val)
    return lowestCommonAncestor(root.right, p, q);
  return root;   // split point or one of them IS root → LCA!
};

// Iterative — O(1) space, same logic
var lowestCommonAncestor = function(root, p, q) {
  let curr = root;
  while (curr) {
    if      (p.val < curr.val && q.val < curr.val) curr = curr.left;
    else if (p.val > curr.val && q.val > curr.val) curr = curr.right;
    else return curr;   // split or match → LCA!
  }
};
// Pattern: BST Split Detection
// Key: The split point = LCA. BST ordering reveals it in O(log n).
⚠️ Gotchas & Common Pitfalls
  • Overcomplicating the search: Don't traverse the whole tree like a standard Binary Tree LCA. Use the BST property!
  • p and q order: p is not guaranteed to be smaller than q. Always check both p < root and q < root instead of assuming p is the min!
  • Missed Equality: The split occurs when p and q are on different sides, OR when the root itself equals p or q. Both mean we found the LCA.
⚖️ Trade-offs & Alternatives
  • Iterative Pointer (Current): O(1) Space, O(H) Time. The absolute best way. Just walks down one single path until the split.
  • Recursive (Alternative): O(H) Space for call stack. Clean code, but unnecessary overhead since it's just tail recursion.
Time: O(h) — single path from root to LCA
Space: O(1) — iterative pointer walk
Pattern: BST split point
ELI5: Two people are walking down a mountain path. As long as they want to go the exact same way (both left or both right), they walk together. The very first fork in the road where one wants to go left and the other goes right—that fork is the Lowest Common Ancestor!
