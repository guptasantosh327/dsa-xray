Validate Binary Search Tree

Step-by-step trace of isValidBST — watch how a valid range [min, max] tightens at every node, catching violations anywhere in the subtree.

Pattern
🎯 Pre-order DFS + Bound Propagation
Pass a valid range [min, max] down to every node. Tighten max when going left, tighten min when going right. Any violation = false immediately.
Same pattern unlocks: #701 Insert into BST · #700 Search BST · #230 Kth Smallest in BST
📋Problem Statement

BST Property: For every node, all values in the left subtree must be strictly less than the node's value, and all values in the right subtree must be strictly greater than it. This must hold for every single node — not just the root!

Given the root of a binary tree, return true if it is a valid BST, false otherwise.

⚠️ The classic trap: a node that looks valid compared to its immediate parent might still violate the BST property due to an ancestor constraint.

Tree [2,1,3] true ✓
Tree [5,1,4,…,3,6] false ✗
Tree [5,4,6,…,3,7] — tricky! false ✗
💡Intuition + Hints
1
Why can't you just compare with parent? Look at this tree: node 3 is in the right subtree of 5 (via node 6), so it MUST be > 5. But 3 < 5 — invalid! Just checking 3 > 6 (its immediate parent) isn't enough — ancestor constraints also apply.
2
The key idea: Pass a valid range [min, max] to every node. A node is valid only if min < curr.val < max.
3
When going left: The new max becomes curr.val — everything left must be smaller. When going right: The new min becomes curr.val — everything right must be larger.
4
Start with validate(root, -Infinity, +Infinity). The root has no ancestors, so it can be anything initially.
5
Use && short-circuit: return validate(left, min, curr.val) && validate(right, curr.val, max). First false stops everything.
isValidBST.js
var isValidBST = function(root) {
  function validate(curr, min, max) {
    if (!curr) return true;               // null = valid base case

    if (curr.val <= min || curr.val >= max)  // bounds violated!
      return false;

    return validate(curr.left,  min, curr.val)  // left: tighten max
        && validate(curr.right, curr.val, max); // right: tighten min
  }

  return validate(root, -Infinity, +Infinity);
};

🧠Mental Model — The Corridor of Valid Values
▸
Think of a narrowing corridor. Every node lives inside a valid range [min, max]. The root's corridor is (-∞, +∞). Each step narrows it — going left shrinks the ceiling, going right raises the floor.
▸
Going left? You set max = curr.val. The entire left subtree must be smaller than where you just came from. The corridor ceiling drops.
▸
Going right? You set min = curr.val. The entire right subtree must be larger. The corridor floor rises.
▸
The tricky case: A node deep in a right-then-left path must satisfy BOTH its parent's constraint AND its grandparent's constraint. The corridor captures this automatically — both bounds travel down together.
🚪

Bounds propagation

The range [min, max] flows top-down. Left child inherits the parent's min but gets a tighter max. Right child inherits the parent's max but gets a tighter min.

⚡

Strict inequalities

BST requires strictly less and greater. Equal values violate the property. So the check is val <= min || val >= max — equals are failures too.

🔁

Pre-order DFS

Root first, then left, then right. We check the current node's validity before recursing. First false anywhere immediately bubbles up through the && chain.

∞

Start with ±Infinity

The root has no ancestor constraints — it can legally be any value. So we start with (-∞, +∞) which every valid integer satisfies.

⚠️Edge Cases — The Gotchas
ancestorAncestor constraint violation: Tree [5,4,6,null,null,3,7]. Node 3 is right child of 6, so 3 > 6 looks… wait, 3 is NOT > 6! But more importantly 3 must be > 5 (root). The bound passed down catches this: min=5 when checking right subtree.
duplicateDuplicate values: A BST cannot have duplicate values — strictly less/greater. So [1,1] is invalid. The <= check catches this.
integersInteger limits: Node values can be Integer.MIN_VALUE or Integer.MAX_VALUE. Don't use those as initial bounds — use -Infinity and +Infinity in JavaScript.
singleSingle node: Always valid — no children to violate the property. validate(root, -∞, +∞) → root.val is always between -∞ and +∞ → true.
✦ Clean solution for revision

var isValidBST = function(root) {
  function validate(curr, min, max) {
    if (!curr) return true;               // null nodes are valid
    if (curr.val <= min || curr.val >= max) return false; // out of corridor
    return validate(curr.left,  min, curr.val)  // left: tighten max
        && validate(curr.right, curr.val, max); // right: tighten min
  }
  return validate(root, -Infinity, +Infinity);
};
// Pattern: Pre-order DFS + Narrowing Bounds
// Key: bounds = corridor of valid values that narrows as we descend
Time: O(n) — each node visited once
Space: O(h) — call stack depth = tree height
Best: O(1) root violates bounds
Worst: O(n) valid tree, full traversal
