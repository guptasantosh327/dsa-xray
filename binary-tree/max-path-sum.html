📋 Problem Statement

Given a binary tree, find the maximum path sum. A path is any sequence of nodes connected by edges where each node appears at most once. The path does not need to pass through the root.

Node values can be negative — a shorter path sometimes beats a longer one!

[-10, 9, 20, null, null, 15, 7] 42 (15→20→7)
All negatives [-3, -1, -2] -1 (best node)
💡 Intuition + Hints
1
Every path bends at exactly one node. At the bend, the path uses both arms: left + curr.val + right. This is the candidate path sum at every node.
2
But only one arm goes upward. When reporting to a parent, a node can only be part of a single-direction path. So it returns curr.val + Math.max(left, right).
3
Prune negatives with Math.max(0, recurse(child)). If a subtree returns negative, treat it as 0 — it's better to stop the path there than to drag the sum down.
4
Track the global answer with let maxSum = -Infinity. Start at -Infinity, not 0, so all-negative trees are handled correctly (the answer must include at least one node).

Core insight: Every node simultaneously plays two roles: as a path peak (evaluates both arms + self, updates global max) and as an arm extender (returns only its best single arm to its parent). Mixing these two calculations is the #1 mistake on this problem.

var maxPathSum = function(root) {
  let maxSum = -Infinity;

  function getMaximumPath(curr) {
    if (!curr) return 0;                          // null node → 0

    let maxLeft  = Math.max(0, getMaximumPath(curr.left))   // ← prune negatives
    let maxRight = Math.max(0, getMaximumPath(curr.right))

    let currMax = curr.val + maxLeft + maxRight;  // path through curr
    maxSum = Math.max(currMax, maxSum);           // update global max

    return curr.val + Math.max(maxLeft, maxRight)  // only ONE arm to parent
  }

  getMaximumPath(root);
  return maxSum;
};

📋What the problem is asking

Given a binary tree, find the maximum path sum. A path is any sequence of nodes where each pair is connected by an edge. The path doesn't need to pass through the root, and each node can appear at most once.

Node values can be negative, so picking a shorter path might give a higher sum than a longer one.

💡Core intuition

At every node, the best path through that node bends here: maxLeft + curr.val + maxRight. But when reporting to the parent, we can only send one arm — the better of left or right.

Use Math.max(0, ...) to prune negative branches — never extend into a subtree that hurts your sum. Track the global max with a closure variable.

🧩Hints to reach the solution
  • 1Think bottom-up: what does each node need to return to its parent? The best single-arm extension: curr.val + max(left, right).
  • 2What does each node compute for itself? The path through it as the peak: left + curr.val + right. Compare this to maxSum.
  • 3Use Math.max(0, recurse(child)) — if a subtree is negative, treat it as 0 (skip it). Never let a bad subtree drag you down.
  • 4Closure variable maxSum = -Infinity captures the answer. Start at -Infinity (not 0!) because all nodes could be negative.
⚡Key trick / pattern

Two roles per node: Every node simultaneously (1) evaluates the path through itself as the peak node (both arms + self), and (2) reports only its best single-arm extension upward. These two jobs need different calculations.

Why -Infinity? If we initialize maxSum = 0, a tree with all negative values would incorrectly return 0. Starting at -Infinity ensures we always capture the actual best node/path.

Why Math.max(0, ...)? If the left subtree returns -5, including it in our path subtracts 5 from our total. The max(0, ...) lets us "opt out" of bad subtrees entirely.

⚠️Edge cases to watch out for
all negAll negative values: The answer is the single largest (least negative) node. Initialize maxSum = -Infinity, not 0. The Math.max(0, recurse) prunes all children so every leaf returns to its parent with just its own value being evaluated.
singleSingle node: maxLeft=0, maxRight=0, currMax=node.val. Even if negative, maxSum updates to that value. Correct — you must always include at least one node.
skewedSkewed/linked-list tree: The path goes straight down one side. At each node one arm is always 0 (null child), so left+curr+right reduces to just the single-direction path. The code handles this naturally.
notePath doesn't need to pass through root: The classic mistake. We update maxSum at EVERY node. The root might not be on the optimal path at all. This is why we use a closure variable, not just return from the root call.
✦ Best solution for learning
O(n) time O(h) space Post-order DFS Closure variable 
var maxPathSum = function(root) {
  let maxSum = -Infinity;               // -Infinity handles all-negative trees

  function dfs(curr) {
    if (!curr) return 0;              // null contributes 0

    let left  = Math.max(0, dfs(curr.left));   // prune if negative
    let right = Math.max(0, dfs(curr.right));

    // This node as the "peak" of the path
    maxSum = Math.max(maxSum, curr.val + left + right);

    // Only one arm goes up to the parent
    return curr.val + Math.max(left, right);
  }

  dfs(root);
  return maxSum;
};

// KEY RULES:
// 1. Math.max(0, recurse) → ignore negative subtrees
// 2. currMax = left + curr + right → evaluate "bent" path at this node
// 3. return curr + max(left, right) → only ONE arm goes to parent
// 4. maxSum starts at -Infinity → handles all-negative inputs
// 5. Update maxSum at EVERY node — answer may not be at root

// how it works

📬

Two jobs per node

Each node does two things: evaluates left + self + right as a complete path candidate, then returns only self + max(left, right) to its parent as one arm.

✂️

Pruning negatives

Math.max(0, recurse(child)) means if a subtree would make the path worse (negative return), we treat it as 0 — as if we simply didn't extend into it.

🎯

The bend node

Every path has one "peak" node where it bends. At that node, both arms are included. We check every single node as a candidate bend, capturing the global max.

🔁

Post-order DFS

Left → Right → Root. Children's values must be known before computing the parent's contribution. Information flows bottom-up through return values.

Time: O(n) — each node visited once
Space: O(h) — call stack height
Init: -Infinity (not 0!)
Pattern: post-order DFS
