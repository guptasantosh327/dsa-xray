Diameter of Binary Tree

Step-by-step trace of diameterOfBinaryTree — watch depth propagate bottom-up and the diameter captured at the right node, not necessarily the root.

Pattern
🎯 Post-order DFS + Global Max Closure
At every node compute leftDepth + rightDepth as a path candidate. Update global max. Return only one arm upward.
Same pattern unlocks: #124 Max Path Sum · #110 Balanced Tree · #687 Longest Univalue Path
📋Problem Statement

Given a binary tree, return the length of the diameter — the longest path between any two nodes. The path length is measured in number of edges.

🔑 The path does NOT need to pass through the root. It can bend at any node. This is the most important insight.

Tree [1,2,3,4,5] diameter = 3
Tree [1,2] diameter = 1
Tree [1] — single node diameter = 0
💡Intuition + Hints
1
Every path has one highest point — a "bend" node where it turns. At that bend, path length = leftDepth + rightDepth. Think of the path as two arms stretching from a single peak node.
2
You don't know which node is the bend in advance. So check every node as a potential bend, keeping a running max.
3
What does a node return to its parent? Its depth — so the parent can compute its own candidate diameter. Return 1 + max(leftDepth, rightDepth).
4
Use a closure variable maxDiameter = 0. Update it at every node. Helper returns a number (depth), not the answer.

Core insight: The function has two roles — it returns depth (for the parent's use) while computing diameter internally (updating maxDiameter). One traversal, two purposes. This separation is the entire trick.

⚡ Critical: The widest path might NOT pass through the root. In the tree [1,2,3,4,5,null,null,6,7], the diameter goes through node 2, not node 1. Always update maxDiameter at every node — not just the root.
diameterOfBinaryTree.js
var diameterOfBinaryTree = function(root) {
  let maxDiameter = 0;                          // single node = 0 edges

  function getDepth(curr) {
    if (!curr) return 0;                        // null → 0 depth

    let leftD  = getDepth(curr.left);
    let rightD = getDepth(curr.right);

    maxDiameter = Math.max(maxDiameter, leftD + rightD); // ← THE KEY LINE

    return 1 + Math.max(leftD, rightD);          // return depth to parent
  }

  getDepth(root);
  return maxDiameter;
};

🧠Mental Model — 30-second recall
▸
Every path bends at exactly one node. At that bend node, path length = left arm + right arm. Your job: try every node as the bend, pick the biggest.
▸
Two different computations at every node: (1) compute leftD + rightD for the diameter candidate, (2) return 1 + max(leftD, rightD) as your depth to the parent. Don't confuse them.
▸
Why closure variable? The helper returns depth (a number). If it also returned the diameter, you'd need to return two values. Easier to capture diameter in an outer variable.
▸
Initialize maxDiameter = 0, not -Infinity. A single node has diameter 0 (no edges). This is correct by default.
🌿

Base case

curr === null returns 0. Leaf nodes get depth 1 because 1 + max(0,0) = 1.

📏

Diameter at a node

leftDepth + rightDepth counts edges of the longest path where curr is the highest (bend) point.

🔝

Why update everywhere

The widest path could bend at any node. We can't know which one, so check at every node and keep the global max.

↑

What gets returned

Returns 1 + max(lD, rD) — depth of this subtree — so the parent can compute its own diameter check.

⚠️Edge Cases
nullEmpty tree → return 0. getDepth(null) returns 0. maxDiameter never updated, stays 0.
singleSingle node → lD=0, rD=0, diameter=0+0=0. Correct — no edges exist.
not rootDiameter does NOT go through root in many trees. Always update at every node, not just root.
edgesleftD + rightD counts edges naturally. No need to subtract 1. Depth measures edges from a node to deepest leaf.

var diameterOfBinaryTree = function(root) {
  let max = 0;
  function depth(n) {
    if (!n) return 0;
    let l = depth(n.left), r = depth(n.right);
    max = Math.max(max, l + r);           // diameter candidate at this bend
    return 1 + Math.max(l, r);             // depth reported to parent
  }
  depth(root);
  return max;
};
// Pattern: Post-order DFS + Closure Variable
// At every node: update max with l+r, return 1+max(l,r)
