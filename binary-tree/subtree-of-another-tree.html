// subRoot tree [4, 1, 2]
isSubtree.js
var isSubtree = function(root, subRoot) {
  const rootSerial    = serializeTree(root)
  const subRootSerial = serializeTree(subRoot)
  return rootSerial.includes(subRootSerial)  // substring check
};

function serializeTree(rootNode) {
  let serialized = ""

  function traversePreOrder(currentNode) {
    if (!currentNode) {
      serialized += '-#'       // null sentinel — marks missing child
      return
    }
    serialized += '-' + currentNode.val  // '-' prefix prevents false matches
    traversePreOrder(currentNode.left)
    traversePreOrder(currentNode.right)
  }

  traversePreOrder(rootNode)
  return serialized
}

📋What the problem is asking

Given two trees root and subRoot, return true if subRoot is a subtree of root — meaning there exists a node in root whose subtree is structurally identical and value-identical to subRoot.

A subtree must match completely. If subRoot has 3 nodes, you need to find a node in root whose exact subtree (including all children and nulls) matches those 3 nodes.

💡Core intuition

Instead of comparing trees node-by-node recursively (the naive approach), we can convert each tree to a string using pre-order traversal. If subRoot's string is a substring of root's string, then it's a subtree.

This works because a valid pre-order serialization uniquely identifies a tree's structure and values. Substring matching gives us an O(m·n) check in a single line: rootSerial.includes(subRootSerial).

🧩Hints to reach the solution
  • 1Write a serialize(node) helper that does pre-order DFS, building a string: "-val" for each node, "-#" for null.
  • 2Serialize both trees. Then just check: rootSerial.includes(subRootSerial).
  • 3The - prefix before every token is non-optional — it prevents false matches like node 12 matching nodes 1 and 2.
  • 4Including -# for nulls encodes the tree's shape — otherwise structurally different trees could produce the same string.
⚡Key trick / pattern

Why the - prefix? Without it, serializing node 12 gives "12" and serializing nodes 1, 2 gives "12" — identical! With the prefix: "-12" vs "-1-2" — now they're distinct. The delimiter goes before each token so every token has a clean left boundary.

Why nulls matter: These two trees look similar but are structurally different:

2 is LEFT child
"-1-2-#-#-#"
≠
2 is RIGHT child
"-1-#-2-#-#"

Without -# sentinels both would serialize to "-1-2" — a false match. The nulls encode shape.

⚠️Edge cases to watch out for
prefixMissing the - prefix: The most common mistake. Without it, node value 12 in root would match a subRoot containing nodes 1 and 2. Always prefix every token.
no nullsSkipping null sentinels: Without -# for null children, structurally different trees can produce the same string. The nulls encode the tree's shape.
empty subsubRoot is null: An empty tree is a subtree of any tree. serialize(null) returns "-#", which always appears inside any root serialization. Returns true.
altRecursive alternative: You can also solve this with two recursive functions — isSubtree traverses root, and for each node calls isSameTree. O(m·n) same complexity, but more code. The serialization approach is more elegant.

var isSubtree = function(root, subRoot) {
  const rootSerial    = serialize(root);
  const subRootSerial = serialize(subRoot);
  return rootSerial.includes(subRootSerial);  // substring check = subtree check
};

function serialize(node) {
  let result = "";
  function dfs(curr) {
    if (!curr) { result += "-#"; return; }  // null sentinel
    result += "-" + curr.val;               // "-" prefix prevents false matches
    dfs(curr.left);
    dfs(curr.right);
  }
  dfs(node);
  return result;
}

// Why this works:
// - Pre-order serialization uniquely identifies a tree's structure + values
// - "-" prefix ensures "-12" ≠ "-1-2" (prevents value-boundary false matches)
// - "-#" for nulls ensures "-1-2-#-#-#" ≠ "-1-#-2-#-#" (shape matters)
// - includes() does a single substring scan — if found, subRoot IS a subtree

📝

Serialize = pre-order string

Visit root → left → right. Each node appends -value. Each null appends -#. The result is a unique string fingerprint of the tree's structure + values.

🛡️

Why the - prefix matters

Without it, node 12 and nodes 1, 2 could produce overlapping strings. The - before every token acts as a delimiter — guaranteeing each token is distinct.

🔍

Substring = subtree

If rootSerial.includes(subRootSerial) is true, the subRoot's exact serialization appears inside root's serialization — meaning it's a real subtree, not just a partial match.

📐

Nulls encode structure

Including -# for null children means trees with different shapes get different strings — even if node values match. Structure and values are both captured.

Time: O(m·n) — string includes() scan
Space: O(m+n) — two serialized strings
Traversal: pre-order DFS
