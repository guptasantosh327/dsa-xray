rightSideView.js 
var rightSideView = function (root) {
    if (!root) return []              // empty tree → nothing to see
    let queque = [root]              // BFS queue seeded with root
    let ans = []

    while (queque.length) {
        let lastNode;
        let len = queque.length;        // snapshot — nodes at THIS level
        for (let i = 0; i < len; i++) {
            let curr = queque.shift()    // dequeue front
            curr.left  && queque.push(curr.left)
            curr.right && queque.push(curr.right)
            if (i == len - 1) {            // last node in this level?
                lastNode = curr            // ← visible from the right!
            }
        }
        ans.push(lastNode.val)
    }
    return ans;
};

📋What the problem is asking

Given a binary tree, imagine yourself standing on the right side of it. Return the values of the nodes you can see, ordered from top to bottom.

At each level (row) of the tree, you can only see one node — the rightmost one. If a node has no right child at some level but a left child exists, that left child becomes visible. You collect exactly one node per level.

💡Core intuition

Use BFS (level-order traversal). Process the tree level by level. At each level, the last node you dequeue is the rightmost — the one visible from the right.

The key insight: BFS always processes nodes left→right within a level. So the last node processed in each level's inner loop is the rightmost. Check i === len - 1 to identify it.

🧩Hints to reach the solution
  • 1Use a BFS queue. Seed it with the root. Process level by level with a while loop.
  • 2At the start of each level, snapshot len = queue.length. This tells you exactly how many nodes belong to this level.
  • 3In the inner for loop (0 to len-1), enqueue children normally. When i === len - 1 (the last node of this level), save its value — that's the right-side-visible node.
  • 4After the inner loop, push the saved value into ans. Repeat for every level.
⚡Key trick / pattern

The len snapshot: let len = queue.length before the inner loop is the backbone of all BFS level-traversal problems. It freezes the count of nodes at the current level. As you process nodes and enqueue their children, the queue grows — but len stays fixed, so you only process this level's nodes.

i === len - 1: The last iteration of the inner loop always processes the rightmost node of that level (because BFS processes left→right). This is simpler than checking "is there a right sibling" or tracking position separately.

DFS alternative: You can also solve this with DFS (right subtree first), taking the first node you see at each depth. BFS is more intuitive for this problem though.

⚠️Edge cases to watch out for
nullEmpty tree: if (!root) return []. Without this guard, queue = [null] and the while loop runs once accessing null.left → crash.
left onlyRight side is not always the right child: Tree — node 3 is the only node at level 2 (left child of 2, right is null). It becomes the rightmost and is visible from the right. Return includes 3. BFS handles this automatically — the last node dequeued at each level is always visible.
skewedSkewed tree (all left children): — every level has exactly one node, so every node is visible from the right. The view equals the entire tree in order: [1, 2, 3, 4].
noteOne node per level guaranteed: Every level has at least one node (otherwise BFS wouldn't process that level at all). So ans.length === treeHeight always.
✦ Best solution for learning

var rightSideView = function(root) {
  if (!root) return [];                  // edge case: empty tree
  let queue = [root], ans = [];

  while (queue.length) {
    let len = queue.length;              // snapshot: nodes at this level

    for (let i = 0; i < len; i++) {
      let curr = queue.shift();          // dequeue front
      if (curr.left)  queue.push(curr.left);
      if (curr.right) queue.push(curr.right);

      if (i === len - 1) {              // last node in this level = rightmost
        ans.push(curr.val);
      }
    }
  }
  return ans;
};

// Key insight: BFS processes nodes left→right within each level.
// The last node processed (i === len-1) is always the rightmost — visible from right.
// len snapshot prevents mixing current and next level nodes in the inner loop.
// Works even when right child is null — the leftmost surviving node becomes visible.

📸

len snapshot trick

Before the inner loop, we save len = queue.length. This tells us exactly how many nodes belong to the current level — even as we add children to the queue.

👁️

Last node = visible node

The condition i == len - 1 fires only for the last node at each level. Since BFS processes left → right, the last one is always the rightmost — exactly what's visible from the right.

🔄

BFS Queue

A queue ensures we process nodes breadth-first. shift() dequeues from front; push() enqueues children to back. Left child before right — preserving left-to-right order.

🎯

One value per level

Each iteration of the while loop handles exactly one level. We collect exactly one value per level — the rightmost one — into ans[].

Time: O(n) — every node visited once
Space: O(n) — queue holds at most one full level
Pattern: BFS level-order
Key check: i == len - 1
