Minimum Depth of Binary Tree

Step-by-step BFS trace of minDepth — watch the queue drain level by level. The instant you hit a leaf, you're done — that's your answer.

🎓 // understanding the problem — read this first!
📏What is Depth?

Depth = the number of nodes on the path from the root to a node. Root itself is depth 1. Its children are depth 2, and so on.

🍃What is a Leaf?

A leaf node = a node with zero children. Both left === null AND right === null. This is the only valid "floor" you can stop at.

⚠️ The Classic Gotcha — don't make this mistake!
If a node has only ONE child (say, left child exists but right doesn't), it is NOT a leaf. You CANNOT stop there. You must go deeper on the side that has a child. This is what makes Minimum Depth harder than it looks.
// correct tree — answer = 2
BFS hits node 9 first (level 2). 9 has no children → leaf found! Answer = 2. We stop immediately, never visiting 20's subtree.
// gotcha tree — answer = 2 (NOT 1!)
Node 1 has a left child (2) but no right child. Node 1 is NOT a leaf! Must go to depth 2 where node 2 is the actual leaf. Answer = 2.
⚡Why BFS is better than DFS for this problem

With BFS you explore level by level. The very first leaf you see is at the minimum depth — stop immediately! With DFS, you'd have to explore the entire tree and track the minimum across all leaf depths. BFS gives an early exit win.

⚡ The only leaf check you need: if (!curr.left && !curr.right) return depth — when BOTH children are null, you found a leaf. Return the current depth immediately. BFS guarantees this is the shallowest possible leaf.

// choose an example

minDepth.js
var minDepth = function(root) {
    if (!root) return 0                       // empty tree → 0

    let queue = [root]                       // seed BFS queue with root
    let depth = 1                              // root itself is depth 1

    while (queue.length) {
        let len = queue.length               // snapshot: nodes at THIS level

        for (let i = 0; i < len; i++) {
            let curr = queue.shift()

            if (!curr.left && !curr.right) {       // ← LEAF CHECK: no children at all
                return depth                    // first leaf = minimum depth!
            }

            if (curr.left)  queue.push(curr.left)
            if (curr.right) queue.push(curr.right)
        }

        depth++                                // done with this level → go deeper
    }

    return depth
};

📋What the problem is asking

Given a binary tree, find the minimum depth — the number of nodes along the shortest path from the root down to the nearest leaf node.

A leaf is a node with no children at all. A node with one child is NOT a leaf — you must continue deeper on that side.

💡Core intuition

Use BFS level by level. At each level, check every node. The very first leaf you encounter is at the minimum depth. Return immediately — BFS guarantees it's the shallowest one.

This is better than DFS because with DFS you'd traverse the whole tree. BFS gives an early exit — sometimes saving a lot of work.

🧩Hints to reach the solution
  • 1Start with standard BFS. Queue seeded with root. depth = 1 (root is level 1).
  • 2Snapshot len = queue.length at each level so you know when to increment depth.
  • 3At each node: check !curr.left && !curr.right. If BOTH are null → leaf! Return depth immediately.
  • 4After the inner loop finishes (all nodes at this level processed), do depth++ and continue.
⚡Key trick / pattern

The critical difference from Max Depth: In max depth you always visit the entire tree. In min depth, BFS lets you stop at the very first leaf. Once found, return depth immediately — don't wait for the level to finish.

Why depth starts at 1: The root node itself counts as depth 1 (it's on the path). If root is a leaf (single node tree), we return 1 immediately.

The one-child trap: A node with one child looks "shallow" but can't be a leaf. BFS naturally handles this — those nodes still enqueue their existing child, so traversal continues.

⚠️Edge cases to watch out for
nullEmpty tree (root = null): Return 0. An empty tree has no nodes, so depth = 0. Handle this before creating the queue.
one childNode with only one child: e.g. tree = [1, 2]. Node 1 has left child 2 but no right. Minimum depth is 2, NOT 1. The check !curr.left && !curr.right (both must be null) correctly skips node 1.
singleSingle node tree: Root is both root and leaf. !root.left && !root.right is true at depth=1. Returns 1 correctly.
noteDFS alternative works too: Recursively, min depth = 1 + min(leftDepth, rightDepth), BUT you must special-case when only one child exists: if left is null, return 1 + rightDepth. The BFS approach avoids this special casing entirely.
✦ Best solution for learning
O(n) time O(n) space BFS level-order Early exit on leaf 
var minDepth = function(root) {
  if (!root) return 0;                     // empty tree

  let queue = [root];
  let depth = 1;                             // root is depth 1

  while (queue.length) {
    let len = queue.length;                 // nodes at this level

    for (let i = 0; i < len; i++) {
      let curr = queue.shift();

      // ← THE KEY CHECK: both children must be null to be a leaf
      if (!curr.left && !curr.right) return depth;

      if (curr.left)  queue.push(curr.left);
      if (curr.right) queue.push(curr.right);
    }

    depth++;                                  // next level
  }

  return depth;
};

// KEY RULES:
// 1. Leaf = !curr.left && !curr.right  (BOTH null, not just one)
// 2. BFS guarantees first leaf found = minimum depth leaf
// 3. Return depth IMMEDIATELY when leaf found (early exit)
// 4. depth starts at 1 (root counts as level 1)
// 5. depth++ only after finishing ALL nodes at the current level

// how it works

🔍

BFS = Level by Level

Queue processes all nodes at depth 1, then all at depth 2, and so on. This guarantees the first leaf you see is at the minimum depth. You never need to go deeper once found.

🍃

Leaf = both children null

The check !curr.left && !curr.right is the entire algorithm's core. One null child → not a leaf, keep going. Both null → leaf, return immediately.

📸

len snapshot

Save len = queue.length before the loop. This tells you exactly how many nodes are on the current level. After processing all of them, depth++.

🚀

Early exit advantage

Unlike max depth where you visit every node, min depth exits the moment the first leaf is found. In best case (balanced tree), you only visit ~half the nodes.

⚠️ Gotchas & Common Pitfalls
  • One-Child Trap: A node with only one child is not a leaf. You must continue down the existing branch. The condition !curr.left && !curr.right ensures BOTH are null.
  • Empty Tree: If root === null, returning 0 right away prevents trying to push null into the queue.
  • Queue Size Snapshot: You must take len = queue.length before processing a level so you know exactly when to increment the depth counter.
⚖️ Trade-offs & Alternatives
  • BFS Iterative (Current): O(W) Space. Often faster than DFS because it immediately exits at the shallowest leaf.
  • DFS Recursive (Alternative): O(H) Call Stack Space. Code is simpler, but it must traverse the entire tree to guarantee it found the absolute minimum depth.
Time: O(n) — worst case all nodes visited
Space: O(n) — queue can hold one full level
Best case: O(n/2) for balanced tree
ELI5: Water is rising from the root level by level. The first node to completely disappear under water (because it has no children below it) defines the minimum depth!
