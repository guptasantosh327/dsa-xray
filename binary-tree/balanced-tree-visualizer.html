Balanced Binary Tree

Pattern
🎯 Post-order DFS + Height Propagation + Early Exit
Return height from each node. Return -1 as a "poison" signal when any subtree is already unbalanced — propagate it upward to skip the rest.

📋 Problem Statement

Given a binary tree, determine if it is height-balanced. A tree is balanced if, for every node, the heights of its left and right subtrees differ by at most 1.

⚠️ The balance condition must hold at every single node — not just the root. A tree can look balanced at the top but be lopsided deep inside.

Balanced — diff ≤ 1 everywhere true ✓
Unbalanced — node 1 has diff = 2 false ✗
💡 Intuition + Hints — Think Before You Code
1
What does every node need to know? To check balance, a node needs the height of both its children. Height = number of edges from that node to the deepest leaf below it.
2
Use post-order (bottom-up). Children must compute their heights first. The parent can't check balance without knowing both children's heights. This is the DFS strategy: go to the leaves first, bubble up answers.
3
The early exit trick — return -1 as "poison". If a child subtree is already unbalanced, why keep checking? Return -1 as a special signal. Any ancestor that sees -1 from a child immediately returns -1 too — propagating the failure upward without any more work.
4
Base case: null returns 0. A null node has height 0 and is trivially balanced. Leaf nodes get height 1 because 1 + max(0, 0) = 1.

Mental model: Imagine each node "reports" its height upward. If a node discovers its children's heights differ by more than 1, it "sounds an alarm" by returning -1. Every ancestor immediately passes the alarm upward. Only if no alarm is ever sounded do we know the tree is balanced.

⚡ The two-role trick: The helper function does two different things at once — it returns height (the number we propagate upward) but also checks balance at every node. The return value -1 is a "poison pill" that short-circuits the entire recursion. Without this trick, you'd need O(n log n) with two separate passes.
isBalanced.js
var isBalanced = function(root) {
  // Returns height if balanced, -1 if unbalanced anywhere below
  function checkHeight(curr) {
    if (!curr) return 0;            // null → height 0, trivially balanced

    let leftH  = checkHeight(curr.left);
    if (leftH  === -1) return -1;   // left already unbalanced → propagate poison

    let rightH = checkHeight(curr.right);
    if (rightH === -1) return -1;   // right already unbalanced → propagate poison

    if (Math.abs(leftH - rightH) > 1) return -1;  // ← BALANCE CHECK

    return 1 + Math.max(leftH, rightH); // return true height to parent
  }

  return checkHeight(root) !== -1;
};

🧠Mental Model — 30-second recall
▸
Post-order is mandatory. You need both children's heights before you can check the current node. Children run first, parent runs last. That's post-order.
▸
-1 is the poison pill. The moment any subtree is unbalanced, return -1. Any ancestor that receives -1 must immediately return -1 without doing any checks. This gives O(n) instead of O(n log n).
▸
Normal return = height. If everything is balanced, return 1 + max(leftH, rightH). This bubbles the height upward so the parent can use it.
▸
Final answer: checkHeight(root) !== -1. If the top-level call returned -1, tree is unbalanced. Anything else means the tree is balanced and that value is the total height.

⚠️Edge Cases
nullEmpty tree → checkHeight(null) returns 0. 0 !== -1 → true. An empty tree is balanced.
singleSingle node → left=0, right=0. |0-0|=0 ≤ 1. Returns 1. 1 !== -1 → true. Always balanced.
deep leftA left-skewed chain of 3 nodes: root→left→left. At root: leftH=2, rightH=0. |2-0|=2 > 1 → returns -1. Unbalanced.
orderCheck left subtree FIRST. If it returns -1, skip the right subtree entirely. This is the early exit optimization.
✦ Clean solution for revision
O(n) timeO(h) spacePost-order DFS-1 poison pill
var isBalanced = function(root) {
  function check(n) {
    if (!n) return 0;                           // null → height 0
    let l = check(n.left);  if (l === -1) return -1; // poison propagation
    let r = check(n.right); if (r === -1) return -1;
    if (Math.abs(l-r) > 1) return -1;             // balance violated
    return 1 + Math.max(l, r);                    // return true height
  }
  return check(root) !== -1;
};
// Pattern: Post-order DFS + -1 poison pill
// checkHeight returns: actual height if balanced, -1 if unbalanced anywhere
🚨Gotchas & Common Pitfalls
▸
O(N^2) Top-Down Trap: Beginners often write two recursive functions: one to calculate height, and one to traverse nodes. Calling height() on every node leads to O(N^2) time. Bottom-up post-order does it in exactly O(N) by passing height upward!
▸
Forgetting to return -1 early: If the left subtree propagates -1, but you don't check for it immediately, you'll still traverse the right subtree. This wastes O(N/2) time in the worst case!
⚖️Trade-offs & Alternatives
▸
Top-Down DFS O(N^2): Easier to write conceptually because height and balance logic are separated. Trade-off: Appallingly slow for skewed trees. Never use this in an interview unless it's just a starting point.
▸
Bottom-Up DFS O(N) (Our Code): Highly efficient, visits every node exactly once. Trade-off: Requires the concept of an overloaded return value where a non-negative number means height and -1 means "poison violation".
ELI5: Imagine hanging weights on a mobile above a baby's crib. If one side hangs much lower than the other, the whole thing tilts. We check every single branch, from the bottom up, to ensure left and right strings differ by 1 at most!
