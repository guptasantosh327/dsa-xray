Trapping Rain Water

Hard DP Prefix Arrays O(n) time O(n) space Single-Pass Build
Pattern
DP Prefix Arrays — maxLeft + maxRight
Water trapped at any index = min(maxLeft[i], maxRight[i]) − height[i]. Pre-computing both boundary arrays in a single bidirectional loop eliminates the inner scan of brute force, dropping time from O(n²) to O(n).
#238 Product of Array Except Self #407 Trapping Rain Water II #84 Largest Rectangle in Histogram
Problem

Given height[] — an elevation map where each bar has width 1 — compute how much water can be trapped after raining.

Think of it as a mountain silhouette seen from the side. Rainwater fills every valley up to the height of the shorter surrounding peak.

Example 1 — classic
height = [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6 — valleys at indices 2,4,5,6,9 trap water
Example 2 — deep valley
height = [4,2,0,3,2,5]
Output: 9 — the large drop after height 4 accumulates quickly
Example 3 — edge: flat roof (no water)
height = [3,3,3]
Output: 0 — equal heights leave no room to trap
Intuition & Hints
  • Brute force O(n²): for each index i, scan left to find its tallest wall, scan right for its tallest wall, compute water. Too slow.
  • Key insight: those max-left and max-right values don't change between queries — pre-compute them once into arrays maxLeft[] and maxRight[].
  • Clever single loop: your solution fills both arrays simultaneously — index i expands the left frontier while nHeight-i-1 shrinks the right frontier, all in one pass.
  • Second pass: a simple linear scan computes min(maxLeft[z], maxRight[z]) − height[z] and accumulates into unitRain.
Key Trick Water trapped = the bucket model: you can only fill up to the shorter of the two walls. min(maxLeft, maxRight) is the water level ceiling. Subtract the bar itself to get the air gap filled with water.
Solution Code
var trap = function (height) {
    let maxLeftHeight = 0;
    let maxRightHeight = 0;
    let maxLeft = []
    let maxRight = []
    let nHeight = height.length
    for (let i = 0; i < nHeight; i++) {
        maxLeftHeight = Math.max(height[i], maxLeftHeight)
        maxRightHeight = Math.max(height[nHeight - i - 1], maxRightHeight)
        maxLeft.push(maxLeftHeight)
        maxRight[nHeight - i - 1] = maxRightHeight
    }
    let unitRain = 0
    for (let z = 0; z < height.length - 1; z++) {
        unitRain += Math.min(maxLeft[z], maxRight[z]) - height[z]
    }
    return unitRain;
};

Gotchas & Common Pitfalls
⚠️
Off-by-one in the water loop: the code uses z < height.length - 1 — it skips the last bar. This is correct because the last bar can never trap water (nothing to its right), but interviewers may probe this. Using z < height.length also works since the last bar always contributes 0.
⚠️
Negative water: min(maxLeft, maxRight) − height[z] can never be negative because maxLeft[z] ≥ height[z] and maxRight[z] ≥ height[z] by definition. But if you accidentally use a plain max rather than running max, you can produce negative values.
⚠️
Edge: all same height — like [3,3,3]. Every position's water = min(3,3)−3 = 0. The algorithm handles this correctly but many beginners forget to verify this edge case in interviews.
Trade-offs & Alternatives

This approach — O(n) / O(n)

Two prefix arrays use O(n) space but give clean, readable logic. Easy to extend or debug. The clever single-loop build is a bonus optimization over two separate passes.

Two Pointers — O(n) / O(1)

Moves left and right inward, advancing the shorter side. No arrays needed — O(1) space. Harder to intuit: you must trust that the current minimum is always the bottleneck wall.

Stack — O(n) / O(n)

Maintains a monotone decreasing stack, computes water horizontally when a taller bar is found. More complex to implement; usually reserved for explaining how water "fills in" layer by layer.

Brute Force — O(n²) / O(1)

For each bar, scan left and right for max walls. Correct but TLEs on large inputs. The DP approach here is essentially brute force with memoization, reducing repeated work.
