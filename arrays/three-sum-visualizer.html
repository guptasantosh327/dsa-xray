Pattern
Sort + Nested Two Pointers
A brute-force 3-loop is O(n³). Sorting first lets you fix one element (target) and reduce the remaining two to a classic Two Sum II — O(n) with converging pointers. Overall: O(n²). Sorting is the unlock that converts a 3D search into a 2D one.
#1 Two Sum #167 Two Sum II #18 4Sum #16 3Sum Closest
Problem

Given an integer array nums, return all unique triplets [nums[i], nums[j], nums[k]] such that they sum to 0. The solution set must not contain duplicate triplets.

Analogy: You're looking for three numbers on a number line that perfectly balance — one pulling left, others pulling right, net force = zero. Sorting lines them up so you can "squeeze" two outer handles to find the balance point.

EXAMPLE 1
Input: [-1, 0, 1, 2, -1, -4] → Output: [[-1,-1,2], [-1,0,1]]
EXAMPLE 2 — all zeros edge case
Input: [0, 0, 0] → Output: [[0,0,0]]
EXAMPLE 3 — duplicates present
Input: [-2, 0, 0, 2, 2] → Output: [[-2,0,2]]
Intuition + Hints
  1. Brute force: three nested loops checking every triplet — O(n³) time, way too slow for n=3000. Can we do better?
  2. Sort first. Once sorted, duplicates are adjacent and easy to skip. More importantly, sorted order makes the inner search steerable — if sum is too big, move right pointer left; if too small, move left pointer right.
  3. Fix one, Two-Sum the rest. Loop over each index as target. For each target, run Two Sum II on the remaining subarray with i and j as converging pointers. This is O(n) per target → O(n²) total.
  4. Skip duplicates. After sorting, if nums[target] === nums[target-1], skip — we'd generate the same triplets again. Likewise skip duplicate i values after a match.
Key Trick Sort + Two-Pointer reduces 3-variable search to 2-variable. Every time sum > 0 you shrink right; < 0 you grow left. Sorted order guarantees you'll never "miss" a valid pair.
Code — Active Recall
var threeSum = function (nums) {
    let ans = []
    nums = nums.sort((a, b) => a - b)
    for (let target = 0; target < nums.length - 1; target++) {
        if (nums[target] !== nums[target - 1]) {
            twoSum(target, nums, ans)
        }
    }
    return ans
};
 
const twoSum = (target, arr, ans) => {
    let i = target + 1
    let j = arr.length - 1
    while (i < j) {
        let sum = arr[target] + arr[i] + arr[j]
        if (sum > 0) {
            j--;
        } else if (sum < 0) {
            i++
        } else {
            ans.push([arr[target], arr[i], arr[j]])
            i++;
            j--;
            while( i < j && arr[i] == arr[i - 1]) i++
        }
    }
}

Gotchas & Common Pitfalls
  • ⚠️
    Forgetting to skip duplicate targets. The condition nums[target] !== nums[target-1] must check previous element, not next. When target=0, nums[-1] is undefined — comparison with any number is true, so index 0 is always processed (safe).
  • ⚠️
    Not skipping duplicate i after a match. After pushing a triplet, i++; j--; — but if nums[i] === nums[i-1], the same triplet would be pushed again. The inner while(arr[i] == arr[i-1]) i++ handles this. Note: j duplicates are automatically avoided because j moves independently.
  • ⚠️
    Using target < nums.length instead of nums.length - 1. If target is the last element, i = j = target+1... wait, no — i = nums.length > j = nums.length - 1, so the while loop exits immediately. It's harmless but wastes a call. Using length - 1 is correct and more expressive.
Trade-offs & Alternatives

Why not HashMap? A HashMap approach would avoid sorting but requires careful deduplication — you'd need a Set of sorted triplets to avoid duplicates, adding overhead and complexity. The sort+two-pointer approach is cleaner and industry-standard for this problem.


Why not keep three nested loops? O(n³) times out for large inputs (n=3000 → 27 billion ops). The sort costs O(n log n) but the two-pointer inner scan reduces O(n²) inner work to O(n) per target → O(n²) total.


Space trade-off: We sort in-place (mutates nums). The output array ans is O(k) where k is the number of triplets. No additional data structures needed — O(1) extra space (ignoring output).
