Problem Statement

Longest Substring Without Repeating Characters

Given a string s, return the length of the longest substring that contains no repeating characters.

Think of a sliding magnifying glass over the string. You keep extending it rightward as long as each new character is unique inside the glass. The moment you see a repeat, you slide the left edge forward — just enough to evict the duplicate — then keep going. Track the widest the glass ever got.
Example 1  |  General case
s = "abcabcbb" → 3  // "abc"
Example 2  |  All duplicates
s = "bbbbb" → 1  // "b"
Example 3  |  Mixed
s = "pwwkew" → 3  // "wke"
Intuition & Hints
  • 1Brute force: Try every substring O(n²), check each for uniqueness O(n) — total O(n³). Too slow.
  • 2Better: Sliding window. Two pointers i (left) and j (right) maintain a candidate window. Only need to know: is s[j] already in the window?
  • 3Key insight: Use a hash set/map for O(1) membership tests. If s[j] is new → add it, expand right, update max. If it's already there → evict s[i], shrink left. Do not advance j yet — check again next iteration.
  • 4Why shrink one step at a time? We might need to remove several characters from the left before the duplicate is gone. Each removal checks: "is the conflict resolved?" before moving right again.
★ Key trick: the else branch shrinks the window from the left without moving j. On the next loop iteration the if check runs again at the same j — only expanding once the conflict is fully resolved.
Code
Your exact solution 
var lengthOfLongestSubstring = function (s) {
    let i = 0;
    let j = 0;
    let max = 0;
    let hash = {}
    while (j < s.length) {
        if (!hash[s[j]]) {
            hash[s[j]] = 1
            max = Math.max(max, j - i + 1)
            j++
        } else {
            delete hash[s[i]]
            i++;
        }
    }
    return max
};

Gotchas & Common Pitfalls
  • ⚠️
    Advancing j during conflict.
    When s[j] is already in hash, you must NOT move j — only evict s[i] and move i forward. The same j gets re-evaluated on the next iteration. Skipping this leads to missed evictions and wrong window size.
  • ⚠️
    Empty string edge case.
    When s = "", the while loop never runs and max stays 0. This happens to be correct — but make sure you're not assuming a non-empty string anywhere else in your reasoning.
  • ⚠️
    truthy check vs explicit undefined.
    !hash[s[j]] treats both undefined and 0 as "not present". Since values are always set to 1 and deleted (never set to 0), this works — but using a Set with has() is semantically cleaner.
Trade-offs & Alternatives

This approach: Delete-based Shrink

Remove one character from left per iteration until conflict is resolved. Simple to reason about. O(n) total — each character is added and deleted at most once. But may take multiple iterations per conflict.

Alternative: Index-store Jump

Store last seen index of each character in the map. When a conflict is found, jump i directly past the previous occurrence: i = map[s[j]] + 1. Fewer iterations, same O(n) but constant factor improvement.

Why not a Set?

A Set works identically here — has() / add() / delete(). The object-as-map approach is equivalent but the Set version is more idiomatic for membership-only tracking.

ASCII-only: int[128] array

If the input is constrained to ASCII (common in interviews), a fixed-size int[128] array with char-code indexing avoids hash overhead entirely. O(1) lookup, cache-friendly, but doesn't generalize to Unicode.
