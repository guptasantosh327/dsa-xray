Easy Frequency Count

#242 · Valid Anagram

Pattern
Frequency Count (HashMap)
Determine if two strings are identical in makeup by iterating one to tally frequencies into a dictionary, and iterating the other to deduct them. If the books balance out to exactly zero without dipping below, it's an anagram!
#49 Group Anagrams #438 Find All Anagrams in a String #383 Find All Anagrams in a String
Problem

Given two strings s and t, return true if t is an anagram of s, and false otherwise. An anagram is a word formed by rearranging the letters of a different word, using all the original letters exactly once.

Analogy: Imagine two identical Lego sets. If you completely disassemble `s` and sort the block types into logical piles, you should be able to build `t` from those exact same piles. No blocks missing, no blocks left over!

Example 1
s = "anagram", t = "nagaram"
→ true
Example 2
s = "rat", t = "car"
→ false
Example 3
s = "ab", t = "a"
→ false (length mismatch)
Intuition & Hints
  • Length Check: The most obvious hurdle. If the string lengths are different, there's a 0% chance they are anagrams. Terminate early.
  • First Loop Additions: Traverse `s`, dumping everything into a Javascript Object (`source`), tallying up the character counts explicitly.
  • Second Loop Deductions: Traverse `t`, deducting counts. If `t` needs a letter that is currently zero (or undefined), it means it's requesting a block that we ran out of—return false immediately!
✦ Note: The algorithm implemented here evaluates source[t[j]] > 0 without explicitly checking undefined. In JS, comparing undefined > 0 cleanly bypasses errors and evaluates directly to false, ensuring safety against missing characters!
Solution Code
javascript
your solution — exact 
var isAnagram = function (s, t) {
    let source = {}
    if(s.length !== t.length) return false
    for (let i = 0; i < s.length; i++) {
        if (source[s[i]]) {
            source[s[i]] += 1;
        } else {
            source[s[i]] = 1
        }
    }
    for (let j = 0; j < t.length; j++) { 
        if(source[t[j]] > 0){
            source[t[j]] -= 1
        }else {
            return false
        }
    }
    return true
};

Gotchas & Common Pitfalls
  • Forgetting the length check: If lengths differ, they can't be anagrams. Skipping this early return wastes cycles!
  • Using two maps: You can build two frequency maps and compare them, but it wastes space and adds a third loop. One map is optimal.
  • Negative values: If `t` has more of a character than `s`, the count dips below 0 or goes `undefined`. Failing to catch this leads to bugs.
Trade-offs & Alternatives

Alternative 1: Sorting
Sort both strings and compare (s.split("").sort().join("") === t...). Takes O(N log N) time, but avoids the hash map overhead. Great for short strings.

Alternative 2: Fixed Array
Use a fixed size-26 integer array. Faster than a Hash Map and explicitly O(1) space, but breaks if the input contains Unicode characters.

ELI5: Deposit characters into a bank account from `s`, then withdraw characters for `t`. If you overdraw, it's not an anagram!
The first loop runs O(S) times, and the second loop runs O(T) times, making the math strictly additive: O(S + T).
* The dictionary `source` size is bounded strictly to the max number of characters in the alphabet (26), so the space doesn't grow towards infinity, bounding it theoretically at O(1) space!
