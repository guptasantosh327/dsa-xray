Easy Iterative Prefix Shrinking

#14 · Longest Common Prefix

Pattern
Iterative Prefix Shrinking
Start with the first string as the longest possible prefix candidate. For each subsequent string, aggressively trim the prefix until it fits — or until nothing remains. Key insight: once a character is eliminated from the prefix, it can never come back. The prefix is monotonically shrinking.
#58 Length of Last Word #28 Find Index of First Occurrence #1143 Longest Common Subsequence
Problem

Find the longest common prefix string among an array of strings. If there is no common prefix, return "".

Analogy: Imagine sorting words in a dictionary. If you open to the "fl" tab, every word there starts with "fl". You are looking for the deepest tab where all your input words appear together.

Example 1
["flower","flow","flight"]
→ "fl"
Example 2
["dog","racecar","car"]
→ "" (none)
Example 3
["interview","inter","internal"]
→ "inter"
Intuition & Hints
  • Brute force: Pick every character position, check if all strings share it — O(S) where S = total chars. Requires careful bounds checking per string.
  • Smarter seed: The first string is the most optimistic prefix guess. Every other string can only make it shorter, never longer.
  • Shrink loop: For each string, check if the current pref matches its start via str.substring(0, prefLength). If not, chop one character and retry. JavaScript safely handles strings shorter than prefLength by returning the whole string — forcing a mismatch and a shrink.
  • Early exit: If prefLength reaches 0, return "" immediately — no common prefix exists.
✦ Key trick: the while loop does not loop over characters one-by-one — it compares the entire prefix string at once using !==. Each iteration removes exactly one character from the end of pref. Worst case per string = O(m) shrinks where m = initial prefix length.
Solution Code
javascript your solution — exact 
var longestCommonPrefix = function(strs) {
    let prefLength = strs[0].length;
    let pref = strs[0]
 
    for(let i = 1;  i < strs.length; i++){
        let str = strs[i]
        while(pref !== str.substring(0, prefLength)) {
            prefLength--;
            if(prefLength == 0 ) return ''
            pref = pref.substring(0, prefLength)
        }
    }
    return pref
};

Complexity
Time
O(S)
S = total characters across all strings
Space
O(1)
Only two scalar variables — no extra structures
Max shrinks
O(m·n)
m = shortest string length, n = # strings
The prefix can only shrink — once a character is removed it never returns. Total work across the whole algorithm ≤ sum of all character comparisons ≤ S.
str.substring(0, prefLength) gracefully handles strings shorter than prefLength — it just returns the full string, which forces a mismatch and a shrink automatically.
Start optimistic (full first string), trim lazily on demand. This greedy approach avoids scanning every character of every string.
