Easy Greedy State Tracking

#1221 · Split Balanced Strings

Pattern
Greedy Counter / State Tracking
For problems checking contiguous balanced states, keep a single balanceCount. Increment for 'L', decrement for 'R'. Whenever the counter returns to exactly 0, you've found a balanced substring! You can greedily cut it right there.
#20 Valid Parentheses #1249 Minimum Remove to Make Valid Parentheses
Problem

Balanced strings are those that have an equal quantity of 'L' and 'R' characters. Given a balanced string s, split it uniformly into some number of balanced strings. Return the maximum number of balanced strings you can obtain.

Analogy: Imagine a seesaw. Every time someone sits on the Left side, it tilts. Same for the Right side. You wait and watch. Every time the seesaw levels out perfectly horizontally, you draw a line and start counting again.

Example 1
s = "RLRRLLRLRL"
→ 4 (RL | RRLL | RL | RL)
Example 2
s = "RLLLLRRRLR"
→ 3 (RL | LLLRRR | LR)
Example 3
s = "LLLLRRRR"
→ 1 (LLLLRRRR)
Intuition & Hints
  • Local vs Global: We don't need a stack. We just need to know the net difference of L's and R's seen so far.
  • Variable Tracking: Use a simple integer balanceCount. Add 1 for 'R', subtract 1 for 'L'.
  • Greedy Cut: As soon as balanceCount === 0, we know the substring ending right here has perfectly matched 'L's and 'R's. Increment your count of completed substrings.
  • Why Greedy? If you have a balanced string "RL", cutting it immediately gives you 1 point. If you merged it with another balanced string "RL" to form a bigger "RLRL", you'd only get 1 point instead of 2. So cut blindly whenever possible.
✦ Key trick: We only care about crossing 0. If it climbs to 5 or drops to -3, we just wait until the moment it returns to 0 to increment the final chunk count.
Solution Code
javascript
your solution — exact 
var balancedStringSplit = function(s) {
    let balanceCount = 0;
    let count = 0
    for(let i = 0; i < s.length; i++) {
        if("R" == s[i]) {
            balanceCount++
        } else {
            balanceCount--
        }

        if(balanceCount == 0){
            count++
        }
    }
    return count
};

Gotchas & Common Pitfalls
  • Forgetting to count when balance reaches 0: You might successfully detect a balanced substring but forget to increment the total count right when it happens.
  • Not acting greedily: Trying to track complex boundaries instead of just splitting the millisecond the balance becomes 0 will cause missed chunks.
Trade-offs & Alternatives

Alternative 1: Stack
You could push 'L' onto a stack and pop when you see 'R' (or vice versa). When the stack is empty, increment the count.
Trade-off: Takes O(N) space instead of O(1) space with an integer counter.

Complexity & Hooks
Time
O(N)

Space

ELI5: Pull a rope +1 for Left, -1 for Right. The second the rope returns completely to dead center (0), you instantly chop it!
All characters are visited exactly once. No complex inner loops or look-aheads needed. Gives O(N).
Because we only increment and decrement a single counter instead of tracking sequences, space is perfectly minimal at O(1).
