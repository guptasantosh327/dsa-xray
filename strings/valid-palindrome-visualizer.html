Easy Two Pointers

#125 · Valid Palindrome

Pattern
Two Pointers (Outside-In)
Start one pointer at the beginning and the other at the end. Move them towards each other, ignoring any non-alphanumeric characters. If at any point they mismatch, it's not a palindrome.
#344 Reverse String #680 Valid Palindrome II #345 Reverse Vowels
Problem

A phrase is a palindrome if, after converting all uppercase letters into lowercase letters and removing all non-alphanumeric characters, it reads the same forward and backward. Alphanumeric characters include letters and numbers. Given a string s, return true if it is a palindrome, or false otherwise.

Analogy: Imagine two people reading the same sentence—one from the left, one from the right. Every time they hit a space or punctuation, they skip it. They must always read the exact same letter aloud until they meet in the middle.

Example 1
s = "A man, a plan, a canal: Panama"
→ true ("amanaplanacanalpanama")
Example 2
s = "race a car"
→ false ("raceacar")
Example 3
s = " "
→ true (empty string reads same)
Intuition & Hints
  • Normalization: Lowercase the entire string upfront to avoid messy capitalization checks later.
  • Pointer Setup: Place pointer i at index 0 and j at index s.length - 1. Proceed while i <= j.
  • Skipping Skips: Instead of looping to skip spaces all at once, if s[i] isn't alphanumeric, just do i++ and continue the `while` loop softly. This is sometimes called "lazy evaluation."
  • Regex check: Checking !s[i].match(/[a-z0-9]/) effectively detects symbols, spaces, and punctuation.
✦ Key trick: We don't allocate a new cleaned string at all. This saves us from an extra O(N) space penalty that we'd have if we did s.replace(/[^a-z0-9]/gi, '') upfront.
Solution Code
javascript your solution — exact 
var isPalindrome = function(s) {
    s = s.toLowerCase()
    let i = 0;
    let j = s.length - 1;
    while(i <= j) {
        if(!s[i].match(/[a-z0-9]/)){
            i++
        } else if(!s[j].match(/[a-z0-9]/)) {
            j--
        } else if(s[i] == s[j]) {
            i++;
            j--;
        } else {
            return false
        }
    }
    return true
};

Complexity
Time
O(N)
N = string length
Space
O(1) *
No new allocations (mostly)
We traverse the string exactly once. Each character is visited at most once by `i` and at most once by `j`.
* Normally O(1), but since we called s.toLowerCase(), that creates a new string taking O(N) space. If we skipped that and used `.toLowerCase()` inside the loop per char, it would be strictly O(1) space!
