Easy Alternating Chunks / Reversal

#541 · Reverse String II

Pattern
Alternating Chunks / Reversal
For every 2k characters starting from the beginning, reverse the first k characters and leave the other k characters as they are. This requires toggling behavior in contiguous blocks across the string.
#344 Reverse String #151 Reverse Words in a String #557 Reverse Words in a String III
Problem

Given a string s and an integer k, reverse the first k characters for every 2k characters counting from the start of the string. If there are fewer than k characters left, reverse all of them. If there are less than 2k but greater than or equal to k characters, then reverse the first k characters and leave the other as original.

Analogy: Imagine reading a book where you read the first k pages backwards, then the next k pages normally, alternating non-stop.

Example 1
s = "abcdefg", k = 2
→ "bacdfeg"
Example 2
s = "abcd", k = 2
→ "bacd"
Example 3
s = "a", k = 2
→ "a"
Intuition & Hints
  • Chunk Extraction: Instead of jumping by 2k, jump by k. This gives you exact bounds for extraction.
  • Boolean Flag: Use a simple boolean pass flag to track whether the current block of size k needs reversal or not.
  • Reverse Mechanism: When pass is true, extract the substring and loop backwards over it to append to the result. When false, append directly.
  • Toggle and Iterate: Toggle the pass boolean on every iteration, and increment your index pointer by k.
✦ Key trick: We don't need complex math for modulo operators. Advancing the loop index strictly by i = i + k and toggling a flag naturally breaks down the string into alternating phases.
Solution Code
javascript
your solution — exact 
var reverseStr = function (s, k) {
    let i = 0;
    let reverse = ''
    let pass = true
    while (i < s.length) {
        let jump = i + k
        let sub = s.substring(i, jump)
        if (pass) {
            for (let j = sub.length -1; j >= 0; j--) {
                reverse = reverse + sub[j]
            }
            pass = false
        } else {
            reverse += sub
            pass = true
        }
        i = i + k
    }
    return reverse
};

Gotchas & Common Pitfalls
  • Array Out of Bounds: When i + k exceeds the string length, Javascript's .substring() handles it gracefully, but in C++ or Java this throws an error. You often need Math.min(i + k, s.length)!
  • Immutable Strings: In JS, strings are immutable. Doing massive string concatenations in a loop (e.g., reverse = reverse + sub[j]) can be slower than using an Array and .join().
Trade-offs & Alternatives

Alternative 1: Array Two Pointers
Convert the string to an array upfront (let arr = s.split('')). Then iterate i += 2*k, and reverse the subarray from i to i + k - 1 in-place using two pointers (left and right), then arr.join('') at the end.
Trade-off: Avoids creating multiple garbage-collected substring chunks and works completely in-place for minimal memory churning.

Complexity & Hooks
Time
O(N)

Space
O(N)

ELI5: It's like a frog! Jump k strides ahead, flip the lilypads behind you, jump k strides ahead, leave them alone. Repeat!
All characters are visited exactly once. The outer loop increments by k and the inner string concatenations span exactly size k. This gives linear time O(N).
Because strings in JavaScript are immutable, building up reverse incrementally takes O(N) auxiliary space.
