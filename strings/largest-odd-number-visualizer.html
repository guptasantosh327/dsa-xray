Easy Right-to-Left Scan

#1903 · Largest Odd Number in String

Pattern
Right-to-Left Scanning
Because the overall parity (odd or even) of a multidigit number only depends on its very last digit, we just need to scan the string backwards starting from the end to chop off any trailing even numbers.
#58 Length of Last Word #67 Add Binary
Problem

You are given a string num, representing a large integer. Return the largest-valued odd integer (as a string) that is a non-empty substring of num, or an empty string "" if no odd integer exists.

Analogy: A giant number's largeness is dictated by its length and its leading digits. To maximize an odd number drawn from the start, we want to include as many leading digits as possible. The only constraint is we must stop just after an odd digit.

Example 1
num = "52"
→ "5"
Example 2
num = "4206"
→ ""
Example 3
num = "35427"
→ "35427"
Intuition & Hints
  • Math Rule: Any number ending in 1, 3, 5, 7, or 9 is odd. Everything preceding it doesn't matter for parity.
  • Greedy Substring: A substring starting at index 0 will inherently generate a larger number than a substring starting at index 1. Thus, we always want index 0 to be the start!
  • Reverse Searching: The trick is to just find the very last odd digit in the entire string. Let's say it's at index k. The substring from 0 to k is logically the largest possible odd configuration.
✦ Key trick: A simple while loop going right-to-left checking num[len] % 2 == 0 gracefully drops trailing even digits in O(N) time with zero extra space logic.
Solution Code
javascript
your solution — exact 
var largestOddNumber = function (num) {
    let len = num.length - 1;
    while (len >= 0 && num[len] % 2 == 0) {
        len--;
    }
    return num.substring(0, len + 1);
};

Gotchas & Common Pitfalls
  • Integer Overflow: You cannot parseInt() the entire string to check if it's odd! JavaScript numbers lose precision after 15 digits. You MUST check the character/string logic by examining the digits directly: num[len] % 2 === 0.
  • Empty Returns: If no odd digits exist (like "4206"), you must return an explicit empty string "", not 0 or null.
Trade-offs & Alternatives

Alternative 1: Regular Expressions
You can use a regex replacement: return num.replace(/[02468]+$/, '').
Trade-off: It's a clean one-liner, but under the hood the Regex engine is still performing a backward scan. Explicitly writing the loop gives you absolute control without regex compiling overhead.

Complexity & Hooks
Time
O(N)
N = string length
Space
O(1)
No auxiliary space needed
ELI5: Eating a chocolate bar where you only like odd pieces! Start biting from the right end until you hit an odd piece. Keep that piece and everything to its left!
In the worst case (e.g. "88888"), we scan the whole string exactly once going backwards, which evaluates in O(N).
The substring slicing takes O(N) allocation space depending on the browser engine under the hood, but in terms of algorithmic tracking space, it's just one integer len making it O(1) extra space!
